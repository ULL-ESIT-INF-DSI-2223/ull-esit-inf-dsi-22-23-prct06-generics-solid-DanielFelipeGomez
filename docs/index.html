Ejercicio 1 - Biblioteca musical

Mediante el uso de clases e interfaces debemos modelar el comportamiento de una biblioteca musical, esta contara con artistas, discos, canciones. Y debemos poder ver datos de las anteriores además de poder buscarlas individualmente.

Artista, Solista y Grupo

Podemos tener dos formas de artistas; solistas y grupos. Ambos deben cumplir el hecho de tener las siguientes propiedades:

  • Un artista tendrá un nombre.

  • Un artista tendrá un número de oyentes mensuales.

  • Un artista tendrá una discografía

    • Una discografía es una colección de discos.

Para poder conseguir estas propiedades tanto el Solistas como en Grupos, he decidido hacer una interfaz Artista que agrupa dichas propiedades y crear dos clases Solista y Grupo que implementan la interfaz mencionada. 

import {Disco} from "./disco";

export interface Artista {
    nombre: string;
    numOyentesMensuales: number;
    discografia: Disco[];
}

He entendido bastante mejor con este ejemplo lo que pretende hacer una interfaz. Al final se puede ver como un contrato donde defino las propiedades que voy a necesitar y al implementarlo en una clase obligo a dicha clase a tener como mínimo las propiedades que se han acordado en al interfaz.

Si ahora nos fijamos con más detalle en las clases Solista y Grupo.

Hice uso de los constructores simplificados para definir los argumentos necesarios para crear un objeto de las clases y a su vez definir el estado de seguridad en el que se encuentran. En este caso yo decidí que sean public todos los datos.

También añadí un método toString con la intención de que sea este método el que se llame a la hora de intentar hacer un console.log() para esto defino un objeto que contiene tres propiedades que a su vez corresponden con las propiedades de la clase.

export class Solista implements Artista {
  constructor(public nombre: string, 
              public numOyentesMensuales: number, 
              public discografia: Disco[]) {
  }
  
  public toString(): object {
    const discos = this.discografia.reduce((acc, elem) =>  elem.toString() + ', ' + acc, '')
    return {
      Nombre: this.nombre, 
      NumeroOyetes: this.numOyentesMensuales,
      Discografia: discos
    };
  } 
}

export class Grupo implements Artista {
  constructor(public nombre: string, 
              public numOyentesMensuales: number, 
              public discografia: Disco[]) {
  }

  public toString(): object {
    const discos = this.discografia.reduce((acc, elem) => acc + ', ' + elem.nombre, '')
    return {
      Nombre: this.nombre, 
      NumOyetes: this.numOyentesMensuales,
      Discografia: discos
    };
  } 
}

Disco

Si nos fijamos en las clases anteriores encontramos una propiedad discografia la cual es un array de Disco este último hay que modelarlo. En mi caso desarrollé una clase Disco que cuenta con tres propiedades.

  • Un disco tiene un nombre.

  • Un disco tiene un año de lanzamiento.

  • Un disco tiene una colección de canciones.

Nuevamente hice uso de los constructores simplificados que nos proporciona el lenguaje. Donde definí todas las propiedades como public. Además, agregue el método toString para conseguir el resultado que antes se mencionaba en las clases Solista y Grupo.

Finalmente mencionar que para poder conseguir los nombres de las canciones en el método toString hice uso de un reduce para concatenar las canciones en una string.

export class Disco {
  constructor(public nombre: string,
              public temporadaPublicacion: number, 
              public canciones: Cancion[]) {
  }

  public toString(): object {
    const canciones = this.canciones.reduce((acc, elem) => acc + ', ' + elem.toString(), '')
    return {
      Nombre: this.nombre, 
      NumOyetes: this.temporadaPublicacion,
      Discografia: canciones
    };
  } 
}

Cancion

Nuevamente, si nos fijamos en la clase Disco podemos ver que tenemos un array de Cancion, para poder modelar el comportamiento de una canción desarrollé una clase. La clase Cancion esta compuesta por las siguientes propiedades:

  • Una canción tiene un nombre.

  • Una canción tiene una duración en segundos.

  • Una canción tiene una lista de géneros.

  • Una canción puede ser un single o no.

  • Una canción tiene un número de reproducciones.

Para poder definir las propiedades anteriores, hice nuevamente un constructor simplificado y definí todas las propiedades como public. Finalmente, definí el método toString de forma similar a como se ha desarrollado en las anteriores clases.

export class Cancion {
  constructor(public nombre: string, 
              public duracionSegundos: number, 
              public generos: string[], 
              public single: boolean, 
              public numReproduciones: number) {
  }

  public toString(): object {
    return {
      Nombre: this.nombre, 
      NumOyetes: this.duracionSegundos,
      Discografia: this.generos.toString(),
      Single: this.single === true ? 'Si' : 'No',
      NumReproducciones: this.numReproduciones
    };
  } 
}

Buscar Artista

Desarrollé un método buscarArtista el cual se le pasa un nombre de artista y lo busca en la lista de artistas. Básicamente se recorre todo el array de Artista buscando una coincidencias con el nombre dado por parámetro y en caso de encontrarse, se retorna. Si no se encuentra se retorna undefined.

Interesante fijarse que aprovecho a definir el arreglo como de Artista de esta forma puedo añadir tanto Solista como Grupo.

public buscarArtista(nombre: string): Artista | undefined {
  for (let i = 0; i < this.artistas.length; ++i) {
    if (this.artistas[i].nombre === nombre) {
      return this.artistas[i];
    }
  }
  return undefined;
}

Buscar Disco

Si deseamos buscar un disco en específico, he implementado un método buscarCancion que dado el nombre del disco, lo busca en toda la lista de discos. Para esto recorro el array de Disco y busco una coincidencia de nombre, en caso de NO encontrarla, retorno undefined

public buscarCancion(nombre: string): Cancion | undefined{
  for (let i = 0; i < this.canciones.length; ++i) {
    if (this.canciones[i].nombre === nombre) {
      return this.canciones[i];
    }
  }
  return undefined;
}

He desarrollado un método numCancionesEnDisco que dado un Disco, determina el número de canciones que componen al disco, para eso basta con comprobar la longitud del array que se usa para agrupar las canciones.

Duración de un Disco

Podemos determinar la duración que tendrá un Disco si contamos el tiempo que tarda cada una de sus canciones. Para esto, he desarrollado un método duracionDisco que dado un Disco recorre cada canción mediante el método reduce y va acumulando la suma de los segundos de cada canción.

public duracionDisco(disco: Disco) {
  return disco.canciones.reduce((acc, elem) => acc + elem.duracionSegundos, 0);
}

Número total de reproducciones de un disco

Si queremos ver cuantas reproducciones tiene en total un disco, esto es comprobar cuantas reproducciones tiene cada canción que lo compone y sumarlas. Para esta tarea desarrollé un método numTotalReproduccionDisco que recorre las canciones con el método reduce y va acumulando la suma de las reproducciones de cada canción.

public numTotalReproduccionDisco(disco: Disco) {
  return disco.canciones.reduce((acc, elem) => acc + elem.numReproduciones, 0);

Ejercicio 2 - Conecta 4

Para el ejercicio 2 deseamos desarrollar el juego de Conectar cuatro, en este tenemos un tablero de 6 x 7 casillas y se introducen piezas por turnos, al final el primer jugador en conseguir poner 4 fichas seguidas de forma horizontal, vertical o diagonal habrá ganado.

Para poder modelar dicho juego en TypeScript, he decidido desarrollar dos clases. Una primera clase Jugador que, como bien su nombre indica, emula el comportamiento de un jugador. Además he desarrollado una clase ConectaCuatro que simula el juego en acción y junta todos los componentes necesarios para jugar.

Jugador

En el juego deberán existir dos jugadores que se irán turnando para poner sus fichas, es por esto que en las partidas deberán existir dos objetos de esta clase.

Constructor y Getter

Para desarrollar la clase Jugador use un constructor simple que cuenta con una sola propiedad _forma esta es privada y es una caracter que se dibujará en la consola para representar las fichas del jugador al que haga referencia. Como deseamos poder acceder a dicha propiedad desde fuera de la clase he desarrollado un getter de _forma.

constructor(private _forma: string) {
}

get forma() {
    return this._forma;
}

Cambiar Forma

Con la idea de darle mayor personalización al juego he desarrollado un método cambiarForma que permite elegir al usuario el caracter que usará en la partida como ficha. Pero solo se le dan 4 opciones, que deberá elegir ingresando un número, si ingresa cualquier caracter fuera del rango se le asignará por defecto un '+' como ficha. Con la limitación del caracter a elegir, se previene que el programa imprima mal los resultados y ayuda a la comprensión por consola.

Finalmente mencionar que la opción de elección se puede pasar como argumento. Sin embargo, en caso de no pasarse se puede elegir mediante un Prompt. Esto lo hice con el fin de simplificar las pruebas y para posibles modificaciones del funcionamiento del programa.

cambiarForma(opcion?: number) {
  if (opcion === undefined) {
    console.log('Selecciona un número para indicar el caracter que quieres usar: \nPresiona 1 para ->  O\nPresiona 2 para ->  X\nPresiona 3 para ->  ©\nPresiona 4 para ->  Ø');
    const prompt = Prompt();
    opcion = parseInt(prompt('>>> '));
  }
  switch (opcion) {
    case 1:
      this._forma = 'O';
      break;
    case 2:
      this._forma = 'X';
      break;
    case 3:
      this._forma = '©';
      break;
    case 4:
      this._forma = 'Ø';
      break;
    default:
      this._forma = '+';
      break;
  }
}

Ingresar Valor

Se pretende jugar dando las instrucciones por línea de comandos, para esto debemos recordar seguir los pasos indicados en la sección de Pasos previos. Debemos pedirle al usuario el número de la columna donde quiere dejar "caer" su ficha, para esto desarrollé el método ingresarValor que recibe como parámetros el Jugador que está en su turno y puede o no recibir la posición de la columna. En caso de no recibirse se le pregunta.

Mediante un bucle while me aseguro que el usuario esta eligiendo una posición viable, es decir que no se sale del rango de las columnas, ni se desborda por las filas.

ingresarValor(jugador: Jugador, posicion?: number) {
  let ocupado = true;
  while (ocupado) {
    const prompt = Prompt();
    if (posicion === undefined) {
      posicion = parseInt(prompt('Introduce el numero donde quieres dejar caer la ficha: ')) - 1;
    }
    while (0 > posicion || posicion > 6) {
      posicion = parseInt(prompt('Introduce el numero donde quieres dejar caer la ficha: ')) - 1;
    }
    let i = 5
    for (i; i >= 0; --i) {
      if (this.tablero[i][posicion] === this.casilla) {
        this.tablero[i][posicion] = jugador.forma;
        ocupado = false;
        break;
      }
    }
    if (i === -1) {
      ocupado = true;
    }
  } 
}

Revisar si alguien ha ganado

Después de que un jugador elige una posición donde poner su ficha debemos comprobar si se ha conseguido ganar. Para esto se debe comprobar lo siguiente:

  • Si hay 4 fichas del mismo jugador seguidas en horizontal. Dicho jugador ha ganado.

  • Si hay 4 fichas del mismo jugador seguidas en vertical. Dicho jugador ha ganado.

  • Si hay 4 fichas del mismo jugador seguidas en diagonal. Dicho jugador ha ganado.

Esta es sin duda la tarea más relevante del juego y la he divido en 4 partes.

Revisar Filas

Vamos a revisar si en una posición dada por parámetros existe una coincidencia de 4 fichas seguidas a la derecha que correspondan al elemento pasado por parámetro.

Básicamente defino un límite izquierdo, esto es que se puedan poner 3 fichas a la izquierda de la posición dada, si esto es posible, significa que hay una potencial victoria.

A continuación se recorre desde el límite izquierdo definido hasta la posición pasada por parámetros. Si se encuentra que un elemento de ese recorrido no coincide con el del elemento pasado, ya se puede descartar y retornar false porque significa que no hay 4 elementos del mismo tipo seguidos.

En caso de recorrer todo el bucle for sin ser descartado, significa que se han recorrido 4 elementos del mismo tipo por tanto se retorna true.

revisarFila(posFila: number, posColum: number, elemento: string) {
  let limIzqFila = -1;
  if (posColum > 2) {
    limIzqFila = posColum - 3;
  }
  if (limIzqFila !== -1) {
    for (let i = limIzqFila; i <= posColum; ++i) {
      if (this.tablero[posFila][i] !== elemento) {
        return false;
      }
    }
    return true
    
  }
  return false;
}
Revisar Columnas

Veamos que esta vez pasará lo mismo que el ejemplo anterior, pero esta vez la parte que se mantendrá estática el la posición de la columna, definiremos un limite esta vez hacia abajo y realizaremos exactamente el mismo procedimiento de antes, con la misma idea.

revisarColumna(posFila: number, posColum: number, elemento: string) {
  let limAbajoColum = -1;
  if (posFila <  3) {
    limAbajoColum = posFila + 3;
  }
  if (limAbajoColum !== -1) {
    for (let i = posFila; i <= limAbajoColum; ++i) {
      if (this.tablero[i][posColum] !== elemento) {
        return false;
      }
    }
    return true
  }
  return false;
}
Revisión de Diagonal a la Izquierda

Para revisar que todo se cumple en la diagonal hacia la izquierda se comprueba primero que desde la posición que se esta evaluando se pueda ganar con una diagonal, esto es que exista la posibilidad de poner 3 fichas más hacia arriba en diagonal.

Si lo anterior se cumple, se evalúa una concatenación de 3 ifs de los cuales desde que alguno no se cumpla se retornará false. Básicamente, se prueba que si accedemos al elemento a la diagonal corresponde con el mismo elemento pasado por parámetros. Si todos los 3 elementos de la diagonal se corresponden al elemento significa que hemos encontrado un ganador y retornamos true.

revisarDiagonalIzquierda(posFila: number, posColum: number , elemento: string) {

Primeramente comprobé que el número ingresado fuera un entero y no fuera negativo. En caso de ser así, retornarñia undefined.

Posteriormente definí mi array bidimensional donde podría las tabals resultantes y definí mediante un bucle while que mienstras contador fuera distinto del tamaño del array + 1 continuará, de esta forma consigo usar el contador para cada uan de las tablas y lo voy incrementando en cada iteración del bucle while.

Luego dentro del bucle whiel use un bucle for donde se calculan las tablas hasta el tamaño del array y se van añadiendo a un array que leugo al terminar el bucle es añadido al resultado.

export function productTable(num: number): number[][] | undefined {
    if (num < 1 || String(num).includes('.')) {
        return undefined;
    }
    const tamArray = num;
    let resultado: number[][] = []
    let tabla: number[] = [];
    let contador = 1;
    while (contador != tamArray + 1) {
        for (let i = 1; i <= tamArray; ++i) {
            tabla.push(i * contador)
        }
        resultado.push(tabla);
        tabla = [];
        ++contador;
    }
    return resultado
}

Simplificar

Los racionales se pueden simplificar a partir de un mínimo común divisor, para conseguir esto. Buscamos el mayor entre el númerador y el dénominador y mediante un bucle for buscamos si existe algún elemento que permita dividir a ambos elementos de forma entera. Esto se repite hasta llegar a probar con 1. 

simplificar() {
  let mayor: number;
  if (this.numerador > this.denominador) {
    mayor = this.numerador
  } else {
    mayor = this.denominador
  }
  for (let i = mayor; i > 0; --i) {
      if (this.numerador % i  === 0 && this.denominador % i === 0) {
          this.numerador = this.numerador / i;
          this.denominador = this.denominador / i;
      }
  }
}

Invertir

Para invertir un número racional, hacemos que el númerador pase a ser el dénominador y el dénominador será ahora el númerador. Para esto simplemente realziamos la títpica operación de swap apoyandonos en una variable auxiliar.

invertir() {
    const aux = this.numerador
    this.numerador = this.denominador;
    this.denominador = aux;
